name
Search
description

This block is used to let users search a site’s content by entering a keyword or phrase, returning relevant pages or documents.

Implementation
This block’s table typically has 1 column and 2 rows, with the first row containing only the block name and optional variants.

  • The second row contains an absolute URL to the query index to search (e.g., https://main--helix-block-collection--adobe.hlx.page/block-collection/sample-search-data/query-index.json)

When rendered, CSS and JavaScript will display a search input field and handle user queries, returning matching results based on your site’s query index.

When to use

  • When you want to give users a quick way to find specific content across your site.
  • On websites with a large volume of pages or documents that users may need to locate quickly.
  • If you want a straightforward, built-in search experience without custom coding.

When to consider another block

  • If your site has minimal or single-page content where searching isn’t necessary.
  • When you prefer an external or more feature-rich custom search solution.
  • If you only need filtered navigation rather than a text-based search.
